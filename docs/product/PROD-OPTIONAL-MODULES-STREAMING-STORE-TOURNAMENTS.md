# PROD — Optional Modules (Streaming / Store / Tournaments)

## Summary
Add modular expansion features once the core club OS is stable, with clean enable/disable per club and clear ROI per module.

## Users
- Club Director
- Master Admin

## Problem
Modules can increase ARPU, but only if they don’t destabilize the core product and are easy to operate.

## Scope
- Module framework (enable/disable per club)
- Candidate modules:
  - Streaming (live practices/games; multi-angle later)
  - Team store
  - Sponsored challenges
  - Tournament management

## Non-goals
- Shipping all modules at once

## Success metrics
- Module attach rate (per tier)
- Incremental retention/ARPU per module

## Acceptance criteria
- [ ] Each module has clear success metric + pricing hypothesis
- [ ] Modules can be enabled/disabled without data corruption
- [ ] Permissions and billing gating apply consistently

## Dependencies
- Monetization/entitlements
- Core features stability

## Open questions
- Which single module is the first ROI bet after core stability?
